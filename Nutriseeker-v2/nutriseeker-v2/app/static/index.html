Architecture (DietAI / NutriSeeker)

  • Frontend: FastAPI-served SPA (this UI)
  • Vision: BLIP-2 (GPU) or BLIP-large (CPU); optional Ollama + LLaVA for richer lists
  • NLP / retrieval: Multilingual sentence embeddings → semantic match to IFCT food names
  • Portions: Keyword priors + optional foodname: 150g in your notes
  • Data: IFCT 2017 CSV (primary) + bundled usda_foods_sample.csv — replace with full USDA bulk export if needed (still no API key)

For development without a local GPU, Google Colab or Kaggle kernels work well; deploy Ollama + LLaVA on a desktop for the stack in your diagram.
